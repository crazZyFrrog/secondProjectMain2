# Amvera: 404 при «Приложение запущено»

Если в логах видно `Uvicorn running on http://0.0.0.0:80`, а в браузере по URL — **404 Not Found**, запрос **часто не доходит до FastAPI** (ответ от nginx / входного контроллера).

## 1. Публичный домен, а не внутреннее имя

По [документации Amvera](https://docs.amvera.ru/applications/configuration/network.html):

- **Внутреннее** имя для связи между проектами в Amvera:  
  `amvera-<проект>-run-<пользователь>` (без `.amvera.io` в примерах для внутренней сети).
- **Публичный HTTPS-домен** создаётся отдельно: **Настройки проекта → Доменные имена → Добавить** → тип **HTTPS**, тип домена **Бесплатный домен Amvera**.

Формат бесплатного домена обычно: **`https://<имя-проекта>.<логин>.amvera.io`** (точное значение показывается в панели после привязки).

Открывайте в браузере **именно тот URL**, который показан в разделе доменов, а не произвольную комбинацию `amvera-*-run-*.amvera.io`, если она не привязана как входной контроллер.

## 2. Порты

В `amvera.yml` должны совпадать:

- `run.containerPort` / `run.servicePort` — **80** (как в нашем Dockerfile и `uvicorn --port 80`).

Если в секретах задан **`PORT`** с другим значением (например `8080`), а приложение слушает **80**, ingress будет стучаться не в тот порт. Либо уберите `PORT`, либо выставьте `PORT=80`.

## 3. Проверка, доходит ли трафик до приложения

После деплоя с актуальным `main.py` в логах при **каждом** запросе к API должна появляться строка:

```text
[HTTP] GET /
```

Если 404 в браузере есть, а **`[HTTP]` в логах нет** — трафик не доходит до контейнера (домен / ingress / порт).

## 4. Полезные пути

- `GET /` → `{"message": "API работает"}`
- `GET /health` → `{"status": "ok"}`
- `GET /docs` — Swagger UI
