# Playwright - Инструкция по запуску тестов

## ✅ Что установлено

- ✅ Playwright (`@playwright/test`)
- ✅ Браузеры (Chromium, Firefox, WebKit)
- ✅ 6 тестовых сценариев на русском языке
- ✅ Вспомогательные утилиты
- ✅ Конфигурация Playwright
- ✅ GitHub Actions workflow (CI/CD)

## 📋 Тестовые сценарии

1. **Регистрация пользователя** (`01-регистрация.spec.ts`)
   - Пользователь регистрируется и попадает в личный кабинет

2. **Создание проекта** (`02-создание-проекта.spec.ts`)
   - Пользователь создаёт проект, он появляется в списке

3. **Валидация формы** (`03-валидация-формы.spec.ts`)
   - Пользователь не может отправить форму с пустым обязательным полем

4. **Доступ без авторизации** (`04-доступ-без-авторизации.spec.ts`)
   - Пользователь без авторизации не видит свои проекты

5. **Удаление проекта** (`05-удаление-проекта.spec.ts`)
   - Пользователь удаляет проект, он исчезает из списка

6. **Редактирование проекта** (`06-редактирование-проекта.spec.ts`)
   - Пользователь редактирует уже существующие проекты

## 🚀 Как запустить тесты

### Шаг 1: Запустите Backend

```bash
cd backend
.venv\Scripts\activate
python -m uvicorn main:app --reload --port 5001
```

Backend должен работать на `http://localhost:5001`

### Шаг 2: Запустите Frontend

```bash
cd frontend
npm run dev
```

Frontend должен работать на `http://localhost:5173`

### Шаг 3: Запустите тесты

```bash
cd frontend
npm test
```

## 🎯 Команды для тестирования

```bash
# Запуск всех тестов (headless)
npm test

# Запуск с видимым браузером
npm run test:headed

# Интерактивный UI режим (рекомендуется)
npm run test:ui

# Режим отладки
npm run test:debug

# Показать отчёт
npm run test:report

# Запуск одного теста
npx playwright test tests/01-регистрация.spec.ts

# Запуск на одном браузере
npx playwright test --project=chromium
```

## 🎨 Интерактивный режим (рекомендуется)

Самый удобный способ запуска и отладки тестов:

```bash
npm run test:ui
```

Это откроет интерактивный интерфейс где можно:
- Выбирать тесты для запуска
- Видеть выполнение шаг за шагом
- Инспектировать элементы
- Смотреть логи и скриншоты

## 🐛 Отладка

Если тесты падают:

1. Запустите в headed режиме чтобы видеть браузер:
   ```bash
   npm run test:headed
   ```

2. Используйте debug режим:
   ```bash
   npm run test:debug
   ```

3. Проверьте что:
   - ✅ Backend работает на `http://localhost:5001`
   - ✅ Frontend работает на `http://localhost:5173`
   - ✅ Тестовый пользователь существует в БД
   - ✅ База данных не повреждена

## 📊 Отчёты

После запуска тестов:

```bash
npm run test:report
```

Откроет HTML отчёт с:
- Результатами всех тестов
- Скриншотами при падении
- Видео записями
- Трейсами для отладки

## 🔧 Настройка

Конфигурация в `playwright.config.ts`:

- Базовый URL: `http://localhost:5173`
- Таймаут: 30 секунд
- Браузеры: Chromium, Firefox, WebKit
- Скриншоты: при падении
- Видео: при падении

## 💡 Советы

1. **Используйте `test:ui`** для разработки тестов
2. **Используйте `test:headed`** для просмотра выполнения
3. **Используйте `test`** (headless) для быстрой проверки
4. **Проверяйте отчёты** после падения тестов
