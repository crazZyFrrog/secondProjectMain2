```bash
npx playwright test --project=chromium
```

## 🎯 Тестовые сценарии

### 0. Проверка настройки ✅
Простой тест для проверки что:
- Frontend доступен на `http://localhost:5173`
- Backend доступен на `http://localhost:5001`
- Страницы загружаются

**Запустите этот тест первым!**

### 1. Регистрация пользователя
- Пользователь заполняет форму регистрации
- Отправляет форму
- Перенаправляется в личный кабинет
- Видит элементы авторизованного пользователя

### 2. Создание проекта
- Пользователь авторизуется
- Создаёт новый проект
- Проект появляется в списке
- Количество проектов увеличивается

### 3. Валидация формы
- Попытка отправить форму с пустым обязательным полем
- Появляется сообщение об ошибке
- Форма не отправляется
- Проверка HTML5 валидации

### 4. Доступ без авторизации
- Неавторизованный пользователь пытается попасть в `/projects`
- Перенаправляется на страницу входа
- Или видит сообщение о необходимости авторизации
- После выхода доступ также закрывается

### 5. Удаление проекта
- Пользователь создаёт тестовый проект
- Удаляет его
- Проект исчезает из списка
- Количество проектов уменьшается
- Проверка отмены удаления

### 6. Редактирование проекта
- Пользователь создаёт проект
- Открывает его для редактирования
- Изменяет название и описание
- Сохраняет изменения
- Изменения отображаются в списке
- Проверка отмены изменений

## 🐛 Отладка

### Если тесты падают:

1. **Проверьте что приложение запущено:**
   ```bash
   # Backend должен быть на http://localhost:5001
   curl http://localhost:5001/docs
   
   # Frontend должен быть на http://localhost:5173
   curl http://localhost:5173
   ```

2. **Запустите тест с видимым браузером:**
   ```bash
   npm run test:headed
   ```
   Вы увидите что происходит в браузере.

3. **Используйте debug режим:**
   ```bash
   npm run test:debug
   ```
   Останавливает выполнение и открывает инспектор.

4. **Проверьте тестового пользователя:**
   - Email: `testforexample@example.com`
   - Password: `password1234`
   - Убедитесь что этот пользователь существует в БД

5. **Посмотрите отчёт:**
   ```bash
   npm run test:report
   ```
   Там будут скриншоты и видео при падении.

## 📊 Отчёты и артефакты

После запуска тестов создаются:

- **`playwright-report/`** - HTML отчёт с результатами
- **`test-results/`** - Скриншоты, видео, трейсы при падении

Чтобы посмотреть отчёт:
```bash
npm run test:report
```

Откроется в браузере с:
- Списком всех тестов
- Статусом каждого теста
- Скриншотами при падении
- Видео записями
- Трейсами для детальной отладки

```bash
npx playwright test tests/01-регистрация.spec.ts
```

### 2. Запуск по паттерну
```bash
npx playwright test --grep "регистрац"
```

### 3. Только на одном браузере (быстрее)
```bash
npx playwright test --project=chromium
```

### 4. Отладка конкретного теста
```typescript
test.only('этот тест', async ({ page }) => {
  // будет запущен только этот тест
});
```

### 5. Пропуск теста
```typescript
test.skip('этот тест', async ({ page }) => {
  // этот тест будет пропущен
});
```

## 🔧 Настройки

Все настройки в `frontend/playwright.config.ts`:

```typescript
{
  baseURL: 'http://localhost:5173',  // URL приложения
  timeout: 30000,                     // 30 секунд на тест
  retries: 0,                         // Локально без повторов
  screenshot: 'only-on-failure',      // Скриншот при падении
  video: 'retain-on-failure',         // Видео при падении
  trace: 'on-first-retry',           // Трейс при повторе
}
```

Можете изменить под свои нужды.
