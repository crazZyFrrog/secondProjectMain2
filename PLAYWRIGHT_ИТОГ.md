## 📝 Тестовые сценарии

### 0. Проверка настройки ✅
- Проверяет доступность frontend (`http://localhost:5173`)
- Проверяет доступность backend (`http://localhost:5001`)
- Проверяет что страницы загружаются

**☝️ Запустите этот тест первым!**

### 1. Регистрация пользователя
- Пользователь регистрируется
- Попадает в личный кабинет
- Видит элементы авторизованного пользователя

### 2. Создание проекта
- Авторизованный пользователь создаёт проект
- Проект появляется в списке
- Количество проектов увеличивается

### 3. Валидация формы
- Попытка отправить форму с пустым полем
- Появляется сообщение об ошибке
- Форма не отправляется

### 4. Доступ без авторизации
- Неавторизованный пользователь не видит защищённый контент
- Перенаправление на страницу входа
- После выхода доступ закрывается

### 5. Удаление проекта
- Пользователь удаляет проект
- Проект исчезает из списка
- Возможность отменить удаление

### 6. Редактирование проекта
- Пользователь редактирует проект
- Изменения сохраняются
- Возможность отменить изменения

## 🚀 Как запустить

### Шаг 1: Запустите приложение

Откройте **2 терминала**:

**Терминал 1 - Backend:**
```bash
cd backend
.venv\Scripts\activate
python -m uvicorn main:app --reload --port 5001
```

**Терминал 2 - Frontend:**
```bash
cd frontend
npm run dev
```

### Шаг 2: Запустите тесты

Откройте **3-й терминал** и выполните:

```bash
cd frontend
npm run test:ui
```

Это откроет интерактивный интерфейс Playwright UI.

### Шаг 3: Запустите первый тест

В UI выберите тест `00-проверка-настройки.spec.ts` и нажмите ▶️ Play.

Если тест прошёл ✅ - всё работает! Можно запускать остальные тесты.

## 📋 Команды

```bash
# В папке frontend:

npm test              # Запуск всех тестов (headless, быстро)
npm run test:headed   # С видимым браузером
npm run test:ui       # Интерактивный UI (РЕКОМЕНДУЕТСЯ)
npm run test:debug    # Режим отладки
npm run test:report   # Показать HTML отчёт

# Запуск одного теста:
npx playwright test tests/00-проверка-настройки.spec.ts

# Только на Chromium (быстрее):
npx playwright test --project=chromium
```

## 🐛 Если тесты не работают

### 1. Проверьте что приложение запущено

```bash
# Backend должен отвечать:
curl http://localhost:5001/docs

# Frontend должен отвечать:
curl http://localhost:5173
```

### 2. Проверьте тестового пользователя

Email: `testforexample@example.com`  
Password: `password1234`

Попробуйте войти вручную в приложение с этими данными.

### 3. Запустите с видимым браузером

```bash
npm run test:headed
```

Вы увидите что именно происходит.

### 4. Посмотрите отчёт

```bash
npm run test:report
```

Там будут скриншоты и видео при падении тестов.

## 📊 Отчёты
