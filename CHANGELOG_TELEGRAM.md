### 📋 Технические детали

#### Когда отправляются уведомления

1. **Регистрация пользователя** (`POST /api/auth/register`):
   ```
   🎉 Новая регистрация пользователя
   
   👤 Имя: Иван Иванов
   📧 Email: ivan@example.com
   🏢 Тип компании: Малый бизнес
   🆔 ID клиента: abc-123
   📦 Текущий тариф: Free
   ```

2. **Переход на Pro тариф** (`PATCH /api/clients/me/plan`):
   ```
   💎 Новый клиент оформил тариф Pro
   
   📦 Тариф: Pro
   📧 Email: user@example.com
   🆔 ID клиента: abc-123
   ```

3. **Переход на Enterprise тариф** (`PATCH /api/clients/me/plan`):
   ```
   👑 Новый клиент оформил тариф Enterprise
   
   📦 Тариф: Enterprise
   📧 Email: user@example.com
   🆔 ID клиента: abc-123
   ```

#### Обработка ошибок
- Все ошибки при отправке уведомлений логируются
- Ошибки **НЕ блокируют** основной сценарий (регистрация/смена тарифа)
- Реализован механизм повторных попыток (3 попытки с экспоненциальной задержкой)
- Таймаут запросов: 10 секунд

#### Безопасность
- Токены бота маскируются в логах
- Все настройки читаются из `.env` файла
- `.env` файл добавлен в `.gitignore` (не коммитится)

### 🔧 Настройка

#### Минимальные требования
- Python 3.7+
- Установленная библиотека `httpx`
- Telegram-бот (создаётся через @BotFather)
- Chat ID получателя (получается через @userinfobot)

#### Переменные окружения
Файл `backend/.env`:
```env
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_MANAGER_CHAT_ID=your_chat_id
```
