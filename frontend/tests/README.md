# Playwright Tests - Landing Constructor

## Описание

Автоматизированные E2E тесты для проекта Landing Constructor, написанные на Playwright.

## Установка

Playwright уже установлен в проект. Если нужно переустановить:

```bash
cd frontend
npm install -D @playwright/test
npx playwright install
```

## Структура тестов

```
frontend/
├── tests/
│   ├── 00-проверка-настройки.spec.ts    # Проверка настройки (запустите первым)
│   ├── 01-регистрация.spec.ts           # Регистрация и вход в личный кабинет
│   ├── 02-создание-проекта.spec.ts      # Создание проекта
│   ├── 03-валидация-формы.spec.ts       # Валидация форм
│   ├── 04-доступ-без-авторизации.spec.ts # Проверка защиты маршрутов
│   ├── 05-удаление-проекта.spec.ts      # Удаление проекта
│   ├── 06-редактирование-проекта.spec.ts # Редактирование проекта
│   ├── helpers.ts                        # Утилиты для тестов
│   └── README.md                         # Эта документация
└── playwright.config.ts                  # Конфигурация Playwright
```

## Запуск тестов

### Перед запуском

Убедитесь, что приложение запущено:

1. **Backend** должен работать на `http://localhost:5001`
   ```bash
   cd backend
   .venv\Scripts\activate
   python -m uvicorn main:app --reload --port 5001
   ```

2. **Frontend** должен работать на `http://localhost:5173`
   ```bash
   cd frontend
   npm run dev
   ```

### Команды для запуска тестов

```bash
# Запуск всех тестов (headless режим)
npm test

# Запуск с видимым браузером
npm run test:headed

# Запуск в UI режиме (интерактивный)
npm run test:ui

# Запуск в режиме отладки
npm run test:debug

# Показать отчёт о последнем прогоне
npm run test:report
```

### Запуск отдельных тестов

```bash
# Запуск конкретного файла
npx playwright test tests/01-регистрация.spec.ts

# Запуск тестов по паттерну
npx playwright test --grep "Регистрация"

# Запуск на конкретном браузере
npx playwright test --project=chromium
```

## Сценарии тестирования

### 1. Регистрация пользователя
- Пользователь регистрируется
- Перенаправляется в личный кабинет
- Видит элементы авторизованного пользователя

### 2. Создание проекта
- Авторизованный пользователь создаёт проект
- Проект появляется в списке
- Количество проектов увеличивается

### 3. Валидация формы
- Попытка отправить форму с пустым обязательным полем
- Проверка сообщений об ошибках
- Форма не отправляется

### 4. Доступ без авторизации
- Неавторизованный пользователь не видит защищённый контент
- Перенаправление на страницу входа
- После выхода доступ закрывается

### 5. Удаление проекта
- Пользователь удаляет проект
- Проект исчезает из списка
- Возможность отменить удаление

### 6. Редактирование проекта
- Пользователь редактирует существующий проект
- Изменения сохраняются
- Возможность отменить изменения

## Отчёты

После запуска тестов отчёт сохраняется в `playwright-report/`. Чтобы посмотреть:

```bash
npm run test:report
```

- Скриншоты (в `test-results/`)
- Видео записи (в `test-results/`)
- Трейсы для отладки

Тесты запускаются на трёх браузерах:
- Chromium (Chrome)
- Firefox
- WebKit (Safari)

Чтобы запустить на одном браузере:

```bash
npx playwright test --project=chromium
```

## Отладка тестов

### UI Mode (рекомендуется)

```bash
npm run test:ui
```

Открывает интерактивный интерфейс с:
- Пошаговым выполнением
- Визуальным отображением действий
- Инспектором элементов

### Debug Mode

```bash
npm run test:debug
```

Запускает браузер с инструментами разработчика.

### Codegen (генерация тестов)

```bash
npx playwright codegen http://localhost:5173
```

Записывает ваши действия в браузере и генерирует код теста.

## Конфигурация

Настройки находятся в `playwright.config.ts`:

- `timeout`: 30 секунд на тест
- `retries`: 0 локально, 2 в CI
- `screenshot`: только при падении
- `video`: только при падении
- `trace`: при первом повторе

## Полезные советы

1. **Используйте `test.only`** для запуска одного теста:
   ```typescript
   test.only('мой тест', async ({ page }) => {
     // ...
   });
   ```

2. **Используйте `test.skip`** для пропуска теста:
   ```typescript
   test.skip('этот тест', async ({ page }) => {
     // ...
   });
   ```

3. **Группируйте тесты с `test.describe`**:
   ```typescript
   test.describe('Группа тестов', () => {
     // тесты здесь
   });
   ```

4. **Используйте хелперы** из `tests/helpers.ts`:
   ```typescript
   import { login, createProject } from './helpers';
   
   await login(page);
   await createProject(page, 'Мой проект');
   ```

## Troubleshooting

### Тесты падают с timeout

- Увеличьте `timeout` в `playwright.config.ts`
- Проверьте, что backend и frontend запущены
- Используйте `test:headed` чтобы увидеть что происходит

### Элементы не находятся

- Используйте Playwright Inspector: `npm run test:debug`
- Проверьте селекторы в браузере
- Добавьте `await page.pause()` для остановки выполнения

### База данных

Убедитесь, что:
- Тестовый пользователь существует в БД
- Backend правильно настроен и работает
- База данных не повреждена
