# 📱 Настройка Telegram-уведомлений

## Шаг 1: Создание Telegram-бота

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Введите название для вашего бота (например, "Landing Constructor Notifications")
4. Введите username для бота (должен заканчиваться на `bot`, например: `landing_constructor_bot`)
5. BotFather пришлёт вам токен в формате: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`
6. **Сохраните этот токен** - он понадобится для `.env` файла

## Шаг 2: Получение Chat ID

### Вариант A: Через специального бота (проще)

1. Найдите в Telegram бота [@userinfobot](https://t.me/userinfobot)
2. Отправьте ему любое сообщение
3. Бот пришлёт вам ваш Chat ID (число, например: `123456789`)
4. **Сохраните этот Chat ID**

### Вариант B: Через API Telegram (альтернатива)

1. Отправьте любое сообщение вашему новому боту (которого создали через BotFather)
2. Откройте в браузере:
   ```
   https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
   ```
   (замените `<ВАШ_ТОКЕН>` на токен из Шага 1)
3. Найдите в JSON-ответе поле `"chat":{"id": 123456789}`
4. **Сохраните это число** - это ваш Chat ID

## Шаг 3: Настройка переменных окружения

1. В папке `backend` создайте файл `.env` (если его нет)
2. Добавьте следующие строки:

```env
JWT_SECRET=your_random_secret_key_here
JWT_ALGORITHM=HS256
JWT_EXPIRES_MINUTES=120

# Telegram bot integration
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
TELEGRAM_MANAGER_CHAT_ID=123456789
```

3. Замените:
   - `your_random_secret_key_here` на случайную строку (можно сгенерировать любым способом)
   - `TELEGRAM_BOT_TOKEN` на токен из Шага 1
   - `TELEGRAM_MANAGER_CHAT_ID` на Chat ID из Шага 2

## Шаг 4: Перезапуск сервера

1. Остановите backend сервер (Ctrl+C)
2. Запустите заново:
   ```bash
   python -m uvicorn main:app --reload --port 5001
   ```

## Шаг 5: Проверка работы

### Проверка через тестовый эндпоинт

1. Войдите как администратор:
   - Email: `admin@example.com`
   - Password: `admin1234`

2. Отправьте POST-запрос на тестовый эндпоинт:
   ```bash
   curl -X POST http://localhost:5001/api/integrations/telegram/test-message \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"text": "Тестовое сообщение из API"}'
   ```

3. Проверьте, что сообщение пришло в Telegram

### Проверка регистрации нового пользователя

1. Зарегистрируйте нового пользователя через frontend (http://localhost:5173/register)
2. В Telegram должно прийти уведомление о новой регистрации:
   ```
   🎉 Новая регистрация пользователя

   👤 Имя: Иван Иванов
   📧 Email: ivan@example.com
   🏢 Тип компании: Малый бизнес
   🆔 ID клиента: abc-123-def
   📦 Текущий тариф: Free
   ```

### Проверка перехода на Pro тариф

1. Войдите под любым пользователем
2. Перейдите в настройки аккаунта и выберите тариф Pro
3. В Telegram должно прийти уведомление:
   ```
   💎 Новый клиент оформил тариф Pro

   📦 Тариф: Pro
   📧 Email: user@example.com
   🆔 ID клиента: abc-123-def
   ```

## Когда отправляются уведомления?

1. **При регистрации нового пользователя** - всегда, при любом тарифе (включая Free)
2. **При переходе на Pro тариф** - когда пользователь меняет тариф на Pro или Enterprise
3. **При переходе с Free на Pro** - также отправляется уведомление

## Устранение проблем

### Сообщения не приходят

1. Проверьте, что файл `.env` находится в папке `backend`
2. Проверьте, что токен бота и Chat ID корректны
3. Убедитесь, что вы отправили хотя бы одно сообщение боту (иначе бот не может отправить вам сообщения)
4. Проверьте логи сервера - там должны быть записи от модуля `telegram_notifications`

### Ошибка "TELEGRAM_BOT_TOKEN и TELEGRAM_MANAGER_CHAT_ID должны быть заданы"

- Убедитесь, что файл `.env` находится в папке `backend` (не в корне проекта)
- Проверьте, что в `.env` нет лишних пробелов вокруг знака `=`

### Бот не отвечает

- Убедитесь, что токен бота корректный
- Попробуйте создать нового бота через BotFather

## Дополнительные возможности

### Отправка в несколько чатов

Вы можете добавить дополнительные Chat ID в переменные окружения:

```env
TELEGRAM_MANAGER_CHAT_ID=123456789
TELEGRAM_ADMIN_CHAT_ID=987654321
```

И использовать их в коде при необходимости отправки в разные чаты.

### Настройка форматирования

В коде используется HTML-форматирование (`parse_mode: "HTML"`). Доступные теги:
- `<b>жирный</b>` - жирный текст
- `<i>курсив</i>` - курсив
- `<code>код</code>` - моноширинный шрифт
- `<a href="url">текст</a>` - ссылка
